# Checkers (Шашки)

Игра в шашки на WPF (.NET 8), реализованная в рамках учебного проекта.

## Возможности

### Режимы игры
- **Игра на двоих** — два игрока на одном компьютере
- **Игра против компьютера** — ИИ на алгоритме Minimax с Alpha-Beta отсечением
- **Продолжить игру** — загрузка сохранённой партии

### Игровая механика
- Стандартные правила шашек
- Обязательное взятие
- Серия взятий одной фигурой
- Дамки — ходят на любое количество клеток по диагонали
- Уведомление когда шашка становится дамкой
- Подсветка доступных ходов
- Победа когда у противника не осталось фигур или ходов

### Интерфейс
- Деревянный стиль доски в классических цветах
- Счётчики съеденных шашек для каждого игрока
- Надпись чей ход
- Цветные уведомления о событиях в игре
- Зелёный цвет при победе, красный при поражении против компьютера

### Дополнительно
- Предложение ничьи — оба игрока должны согласиться
- Против компьютера — ИИ соглашается на ничью если проигрывает по очкам
- Кнопка "Сдаться" для досрочного завершения игры

### Сохранение
- Автосохранение при закрытии окна игры
- Выбор папки для сохранения
- Два формата: **JSON** и **XML**
- Переключение между форматами с копированием данных
- Проверка корректности файла сохранения
- Папка сохранения запоминается между запусками

## Алгоритм ИИ

Компьютер использует алгоритм **Minimax с Alpha-Beta отсечением** глубиной 5 ходов:

- На своих ходах выбирает максимально выгодный ход
- На ходах противника предполагает оптимальную игру
- Alpha-Beta отсечение убирает заведомо невыгодные ветки дерева
- Оценочная функция считает количество фигур на доске
- Соглашается на ничью если у него меньше очков (шашка = 1, дамка = 3)

## Принципы ООП

- **Абстракция** — `Piece`, `GameSerializerBase`
- **Наследование** — `Checker` и `King` наследуют `Piece`, `GameVsComputer` наследует `Game`
- **Полиморфизм** — переопределение `AvailibleMoves` и `AvailibleCaptures` в `Checker` и `King`
- **Инкапсуляция** — приватные поля и методы во всех классах
- **Интерфейсы** — `IMovable` для фигур, `IGameSerializer` для сериализаторов
- **Делегаты** — `Action<>` для событий `OnGameOver`, `OnKingPromotion`
- **Перегрузка операторов** — `==` и `!=` для `Piece`
- **Перегрузка методов** — `Select(pos)` и `Select(pos, bool)` в `Game`
